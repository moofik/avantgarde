# Avantgarde Audio Core --- Полный флоу работы

Документ описывает архитектуру и порядок выполнения: 1. AudioEngine 2.
ClipTrack (MVP)

------------------------------------------------------------------------

# 1. AudioEngine

## Общая роль

AudioEngine --- центральное RT‑ядро движка.

Он: - управляет треками - принимает RT-команды - синхронизируется с
ParamBridge - синхронизируется с TransportBridge - вызывает RT
extensions - микширует треки в master - передаёт master в RecordSink

Никаких аллокаций и блокировок в processBlock.

------------------------------------------------------------------------

## Приватные поля

### tracks\_

Хранит все зарегистрированные треки. Используется только вне RT при
регистрации.

------------------------------------------------------------------------

### rtQueue\_

RT-safe очередь команд.

Назначение: - Control thread пушит команды - RT thread читает команды в
processBlock

------------------------------------------------------------------------

### paramBridge\_

Double-buffer механизм параметров.

RT: - swapBuffers() вызывается строго в начале блока

------------------------------------------------------------------------

### transport\_

Глобальный музыкальный транспорт.

RT: - swapBuffers() - advanceSampleTime()

------------------------------------------------------------------------

### rtExt\_

RT extensions (секвенсор, квантизация и т.п.)

------------------------------------------------------------------------

### masterSink\_

Post-master tap. Позволяет записывать мастер-выход.

------------------------------------------------------------------------

### numOut\_

Количество выходных каналов. Используется для очистки master перед
миксом.

------------------------------------------------------------------------

# Порядок выполнения processBlock

1.  Очистка master out\
2.  Drain RT queue\
3.  ParamBridge.swapBuffers()\
4.  Transport.swapBuffers()\
5.  RtExtension.onBlockBegin()\
6.  Track.process()\
7.  RtExtension.onBlockEnd()\
8.  Transport.advanceSampleTime()\
9.  RecordSink.writeBlock()

------------------------------------------------------------------------

# 2. ClipTrack (MVP)

## Общая роль

ClipTrack --- аудиотрек, воспроизводящий семпл.

Поддерживает: - 1 слот - Play / Stop - Gain - Loop

------------------------------------------------------------------------

## Внутренние сущности

### ClipBuffer

Хранит декодированный wav.

------------------------------------------------------------------------

### clipCtl\_

Control-side владелец памяти. Гарантирует, что RT не получит висячий
указатель.

------------------------------------------------------------------------

### pendingClip\_

Atomic указатель. Сигнал RT: применить новый клип.

------------------------------------------------------------------------

### pendingClear\_

Atomic флаг очистки слота.

------------------------------------------------------------------------

### pendingLoop\_

Atomic флаг изменения loop. 0xFFFFFFFF = нет обновления.

------------------------------------------------------------------------

### RtState

Состояние, живущее исключительно в RT: - clip - playhead - gain -
playing - loop

------------------------------------------------------------------------

# Как работает loadSlotFromFile

1.  Декодируется wav (control thread)
2.  Создаётся ClipBuffer
3.  Сохраняется в clipCtl\_
4.  Указатель публикуется в pendingClip\_

RT увидит его на границе блока.

------------------------------------------------------------------------

# rtApplyPending\_()

Вызывается в начале process.

Применяет: - pendingClear\_ - pendingClip\_ - pendingLoop\_

Это единственная точка перехода control → RT.

------------------------------------------------------------------------

# ClipTrack.process()

Алгоритм:

1.  rtApplyPending\_
2.  Если playing == false → return
3.  Для каждого фрейма:
    -   читать sample
    -   умножить на gain
    -   добавить в out
    -   инкремент playhead
    -   если конец: если loop → reset иначе stop

------------------------------------------------------------------------

# Потоковая модель

Control thread: - loadSlotFromFile - setLoop - armRecordSlot

RT thread: - process - onRtCommand - rtApplyPending

Никаких shared_ptr или аллокаций внутри RT.

------------------------------------------------------------------------

# Архитектурная философия

AudioEngine --- оркестратор.\
ClipTrack --- источник сигнала.\
Transport --- глобальное время.\
RtExtension --- модификатор поведения.\
RecordSink --- выходной tap.

Все границы потоков проходят только через atomic-поля или lock-free
очереди.
